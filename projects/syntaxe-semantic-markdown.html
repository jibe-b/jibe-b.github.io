Syntaxe Semantic Markdown

Main idea

The semantic Markdown version derives from John Gruber's Markdown version, to which semantic markups are added. They correspond to html+RDFa markups, except that the embedding in the text does not correspond exactly to the RDFa embedding.

Reserved characters are used to embed the semantic markups, intending to represents graphs (two nodes, one link), as the minimal structure.

Semantic Markdown is intended to enable easy on-the-fly writing of text with RDFa embedded and therefore to author easily RDF triples.

Reserved characters

@ § % as well as < (followed by a space) and > (preceeded by a space)

The short syntax for hyperlinks is not modified compared to John Gruber's

Markdown version, but a space is required after < and before >.

Basic syntax

Syntax

syntax caracter value example1 example2 Note

base @ subject @molecule @http://dbpedia.org/resource/Molecule

base § predicate §isMadeOf

base % object %atom @http://dbpedia.org/resource/Atom

base " label(beginning and end) @atom"Atom"

base < triple beginning < A @molecule... must be followed by a space, otherwise considered as a not semantic markup

base > triple end %atoms > must be followed by a space, otherwise considered as a not semantic markup

base - follows short namespace name @dbpedia-atom

base : follows predicate and preceeds value in datavalue predicate §hasAge:50

base ^ datatype in datatype predicate §hasAge:50^integer not totally defined

What the basic syntax enables

Basic syntax enables:



- simple triples - ex: _< @ § % >_ and _< @ §: >_
  • multiple element triple, as long as all these elements will be connected with one another - ex: < @ @ § § § % % > (all subjects are linked to all objects by all the predicates) and < @ § % § % > (the subject is linked to the two objects by the two predicates. Multiple syntax is required to link the subject to the two objects separately (by one predicate for each object))

Multiple triple syntax

Syntax

syntax caracter value example1 example2 Note

mult << multiple triple beginning

mult >> multiple triple end

What the multiple triple syntax enables

Multiple triple syntax enables:

  • using a subject (or predicate, or object) in several triples, where each triple is made of predicates and objects not shared between the triples. - ex:

<< @ < § % > < § % %> < §: > < @ § %> >> (the last subject (@) is added to the first subject like if it was written < @1 @2 § % >.

Lines differentiation

In combination with either (or combined) basic and multiple triple syntax,

disctinction can be made between lines containing semantic markup and lines containing text.

Even lines must contain semantic markups and odd lines must contain text (with little markup).

Triples in the even lines are written as in the syntax that makes no distinction between lines,

except that alternative names (also called labels) can be markuped in the text.

Syntax

syntax caracter value example1 example2 Note

lines <<< even/odd syntax section beginning not totally defined

lines >>> even/odd syntax section end not totally defined

In the line containing text, markups must be put but the following text will not be read as subject,

predicate or object but as alternative name.

Even and odd lines

Even lines = lines with even number, with the line where <% is as line zero.

Odd lines = lines with even number, with the line where <% is as line zero.

Example

<<< < @dbpedia-Molecule §isMadeOf %dbpedia-Atome >

< A @molecule §"is made of" %atoms. >

In this case, the label of the object is "atoms" as the triple is equivalent to

< A @dbpedia-Molecule"molecule" §isMadeOf"is made of" %dbpedia-Atom"atoms". >

Recommandation

It is recommended to adopt the even/odd syntax as soon as there are several triples in the text. This

is due to readability considerations.
