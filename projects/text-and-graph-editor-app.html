Text and Graph editor app

Another way of writing.

Context

The future (at least the close future) of data storing and data use are Linked Data. May you be convinced or not, it could be right.

Still today, we do write, store and use what we write, but Linked Data is rarely present.

Main idea

Why not writing documents that contain Linked Data right into them, without the difficult step of coding RDF/XML triples?

Writing in Markdown using semantic markups makes it possible to embed usable triples right into any text.

And as RDF has by essence a graph structure, it is possible to create graphs that are stored in semantic markdown and/or in RDF/XML and/or in JSON.

Storing both text and graph in the same format would enable to synchronise what is produced in both formats. And to write using both methods: graph edition and text edition.

Objectives

  • make semantic markdown convertible to html+RDFa
  • create an app for editing graphs
  • make the synchronisation between text and graph editing clean, i.e.
    • make semantic markdown and the graph format compatible
    • store text and graph in a clean way
    • avoid graph editing from making it totally dirty and absolutely not user-friendly.
